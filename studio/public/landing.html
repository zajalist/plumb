The problem

Foundation models hallucinate physically impossible space — with full confidence.

Trained on text, not gravity. Asked to reason about layout, proximity, support, or kinematics, an agent emits coordinates that look plausible and break on contact. Without local validation it round-trips to the engine and burns context tokens guessing until something looks right.

Propose · validate · repair · commit

A grounding loop, not a better prompt.

Every proposed change runs a gauntlet of gates and halts at the first hard failure. A failed gate returns its margin and a fix vector — so the agent converges in a couple of steps with hard numbers, locally, before anything touches the engine.

The mental model

Three nouns, one loop.

PLUMB caches what each object is, lets you author what you want, and proves what is true — every time, before commit.

Profiles

What each object is.

Bake an asset once into a Physical Asset Profile — geometry, composition, mass, centre of mass, inertia, affordances, rest states. Cached, versioned, human-overridable.

CoACD parts → density-weighted mass / CoM / inertia.
Constraints

What you want.

Author intent — artistic and scientific — in a visual node graph. Each constraint compiles to a cost function with a tolerance: facing, clearance, keep-clear, support.

Soft costs guide repair; hard laws gate the commit.
Gates

What's true.

Every change runs the gauntlet — Collision, Stability, Constraints, Reachability — and halts at the first hard failure, returning the margin and the gradient to fix it.

Never a bare no — always why, and which way.
Type-aware bake

A hollow glass vase and a solid bronze figure are physically nothing alike.

To a naive geometry pass they're identical. PLUMB bakes composition — convex decomposition, per-part materials, density-weighted mass and a centre of mass that actually sits where the weight is. That's the difference between a stack that holds and one that topples.

Bake is the moat.

Each asset is decomposed into convex parts, materials are guessed then human-confirmed, and the physical profile is computed once and cached. Bakes are type-aware: a door, a tree, and a shelf get different recipes — a door bakes a swept hinge volume, a shelf bakes fill regions.

glTF and USD describe geometry. .wdf describes meaning, physics, and intent.

The world language

It all compiles to one portable file.

A .wdf is a declarative language for physically-grounded worlds: the assets, their profiles, the intent graph, and the laws — round-trippable, diff-able, and engine-agnostic.

the_gallery.wdfround-trip
# the_gallery — a physically-grounded scene
import "gallery.vocab"

asset bronze_figure {
  profile   rigid_prop
  mass      48.2kg
  com       [0.0, 0.0, 0.71]
  material  body=bronze base=stone
}

place bronze_figure on pedestal_a {
  at       [0.30, 0.0, 0.40]   # too close to the edge
  law      stability(margin >= 0)   # hard — gates the commit
  law      facing(toward=entrance)  # soft — guides repair
}
# validate → stability −7cm · suggest_transform +6cm → commit ✓
Who it's for

One engine, one language, many builders.

The substrate doesn't care what the laws mean, so the same gates and the same .wdf serve very different teams.

Game studios · level design

AI set-dressing you can trust.

Arrange scenes that are guaranteed stable, clip-free, and navigable, instead of hand-fixing every AI placement.

AI & agent developers

Spatial validation as one tool call.

The agent asks "is this valid, and how do I fix it" and converges in a step or two, instead of burning tokens guessing.

Simulation & robotics

Valid training scenes at scale.

Mass-produce physically valid layouts that hold up the instant physics runs.

Architecture & accessibility

Auditing with an auto-fix.

Ask "is this path wheelchair-wide?", get a pass or fail with the margin, and an automatic correction.

Nothing commits until it hangs true.

A plumb line is the oldest tool for testing whether something hangs true under gravity. PLUMB does that for an agent's spatial decisions — propose, validate, repair, commit.
